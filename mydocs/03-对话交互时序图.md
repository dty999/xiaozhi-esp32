# 小智AI机器人 对话交互时序图

> 此图展示从唤醒词触发到完整一轮对话结束的交互流程

## 正常对话流程（唤醒 → 采集 → 识别 → 推理 → 合成 → 播放）

```mermaid
sequenceDiagram
    participant USER as 👤 用户
    participant WAKE as 🎤 唤醒词检测<br/>ESP-SR
    participant APP as 📱 Application<br/>主循环
    participant AUDIO as 🔊 AudioService<br/>音频服务
    participant PROTO as 📡 Protocol<br/>通信协议
    participant SRV as ☁️ 云端服务器
    participant AI as 🧠 ASR→LLM→TTS<br/>AI 管道

    Note over USER,AI: ═══ 阶段一：唤醒 ===================================
    USER->>WAKE: "小智小智"
    activate WAKE
    WAKE->>APP: 铃2: WAKE_WORD_DETECTED<br/>[音频线程 → 事件组]
    APP->>AUDIO: EncodeWakeWord()<br/>将唤醒词编码为 OPUS
    APP->>APP: SetDeviceState(kDeviceStateConnecting)
    APP->>APP: HandleStateChangedEvent()<br/>UI显示"连接中"

    Note over APP,SRV: ═══ 阶段二：打开音频通道 ===========================
    APP->>PROTO: ContinueWakeWordInvoke()
    PROTO->>SRV: OpenAudioChannel()<br/>建立双向音频流
    SRV-->>PROTO: 通道建立成功
    PROTO->>APP: 铃回调: OnAudioChannelOpened
    APP->>APP: SetListeningMode(AutoStop)<br/>进入 Listening 状态
    APP->>APP: HandleStateChangedEvent()<br/>UI显示"正在听"
    APP->>AUDIO: EnableVoiceProcessing(true)<br/>启动麦克风采集+编码
    APP->>AUDIO: PlaySound(OGG_POPUP)<br/>播放"叮咚"提示音

    Note over USER,AI: ═══ 阶段三：用户说话 → 实时采集编码 → 发送服务器 ==
    USER->>WAKE: "今天天气怎么样"
    AUDIO->>AUDIO: 麦克风采集 PCM 音频
    AUDIO->>AUDIO: OPUS 编码压缩
    AUDIO->>APP: 铃1: SEND_AUDIO<br/>编码包就绪

    loop 持续发送音频包
        APP->>AUDIO: PopPacketFromSendQueue()
        APP->>PROTO: SendAudio(packet)
        PROTO->>SRV: 音频数据包（OPUS格式）
    end

    Note over SRV,AI: ═══ 阶段四：服务器端 AI 处理 =========================
    SRV->>AI: 转发 OPUS 音频流
    AI->>AI: ASR 语音识别<br/>→ "今天天气怎么样"
    AI->>SRV: JSON: {"type":"stt","text":"今天天气怎么样"}
    SRV-->>PROTO: 下发识别结果
    PROTO->>APP: OnIncomingJson回调
    APP->>APP: 屏显 >> 今天天气怎么样<br/>[用户说的话]

    AI->>AI: LLM 大模型推理<br/>→ "今天北京晴，20~28度..."
    AI->>SRV: JSON: {"type":"llm","emotion":"sunny"}
    SRV-->>PROTO: 下发表情指令
    PROTO->>APP: OnIncomingJson回调
    APP->>APP: SetEmotion("sunny")<br/>显示晴天表情

    AI->>AI: TTS 语音合成<br/>→ 逐句生成 OPUS 音频
    AI->>SRV: JSON: {"type":"tts","state":"sentence_start","text":"今天北京晴..."}
    SRV-->>PROTO: 标注句子开始
    PROTO->>APP: OnIncomingJson回调
    APP->>APP: SetDeviceState(kDeviceStateSpeaking)<br/>UI显示"正在说"
    APP->>APP: 屏显 << 今天北京晴...<br/>[小智说的话]

    Note over USER,AI: ═══ 阶段五：音频流下发 → 解码播放 ==================
    loop 服务器下发合成音频
        SRV->>PROTO: OPUS 音频数据
        PROTO->>APP: OnIncomingAudio回调
        APP->>AUDIO: PushPacketToDecodeQueue()
        AUDIO->>AUDIO: OPUS 解码
        AUDIO->>USER: 🔊 扬声器播放
    end

    AI->>SRV: JSON: {"type":"tts","state":"stop"}
    SRV-->>PROTO: 标注合成结束

    alt AutoStop模式（默认）
        PROTO->>APP: OnIncomingJson回调
        APP->>APP: SetDeviceState(kDeviceStateListening)<br/>自动继续收音
        Note over APP,USER: 可继续对话，无需再次唤醒
    else ManualStop模式
        PROTO->>APP: OnIncomingJson回调
        APP->>APP: SetDeviceState(kDeviceStateIdle)<br/>一轮对话结束
        Note over APP,USER: 需重新唤醒才能再对话
    end

    WAKE-->>APP: 若用户不说话 → VAD检测静音
    APP->>PROTO: CloseAudioChannel()

    deactivate WAKE
```

## 对话状态切换一览

| 状态 | 触发条件 | 行为 |
|------|----------|------|
| **Idle → Connecting** | 唤醒词 / 按键 | UI显示"连接中"，打开音频通道 |
| **Connecting → Listening** | 通道建立完毕 | 启动麦克风采集，发送OPUS音频 |
| **Listening → Speaking** | 服务器发来 `tts/state=start` | 停止采集，开始播放服务器音频 |
| **Speaking → Listening** | 服务器发来 `tts/state=stop` (AutoStop) | 自动恢复采集 |
| **Speaking → Idle** | 服务器发来 `tts/state=stop` (ManualStop) | 对话结束 |
| **Listening → Idle** | 按键关闭 / VAD静音超时 | 关闭音频通道 |
